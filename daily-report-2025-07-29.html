🔧 기술적 구현 사항

사용자 관리 시스템

새로 생성된 파일:
  • /app/(dashboard)/users/new/page.tsx - 사용자 생성 페이지
  • /app/(dashboard)/users/[id]/edit/page.tsx - 사용자 편집 페이지
  • /scripts/create-system-user.ts - 시스템 사용자 생성 스크립트

감사 로그 복구 작업

수정된 API 파일:
  • /app/api/products/route.ts - 제품 생성 감사 로그
  • /app/api/products/[id]/route.ts - 제품 수정/삭제 감사 로그
  • /app/api/products/[id]/inventory/route.ts - 재고 업데이트 감사 로그
  • /app/api/brands/route.ts - 브랜드 생성 감사 로그
  • /app/api/brands/[id]/route.ts - 브랜드 수정/삭제 감사 로그
  • /app/api/users/route.ts - 사용자 생성 감사 로그
  • /app/api/users/[id]/route.ts - 사용자 수정 감사 로그
  • /app/api/orders/route.ts - 주문 생성 및 재고 감소 감사 로그
  • /app/api/orders/[id]/status/route.ts - 주문 상태 변경 및 재고 복원 감사 로그

🛠️ 주요 해결 사항

외래 키 제약 조건 오류 해결

AuditLog 테이블의 userId 외래 키 제약 조건 위반 문제를 시스템 사용자('system') 생성으로 해결했습니다.

// 시스템 사용자 생성 const systemUser = await prisma.user.create({ data: { id: 'system', name: 'System', email: 'system@kfashion.com', role: 'MASTER_ADMIN', status: 'ACTIVE', emailVerified: new Date(), }, })

모든 API 경로 감사 로그 표준화

모든 API 경로에서 일관된 패턴으로 감사 로그를 생성하도록 표준화했습니다.
